> ### 角色与任务
> 
> 
> 你是一个高级全栈桌面应用开发工程师。请帮我使用 **Electron** 框架开发一个轻量级的 Windows 本地音频剪辑软件。
> ### 技术栈要求
> 
> 
> * **应用框架**: Electron (主进程与渲染进程分离，通过 IPC 通信)。
> * **前端 UI**: 原生 HTML + CSS (采用 Flexbox/Grid 自适应布局) + Vanilla JS。
> * **波形可视化与播放**: `wavesurfer.js` (及其 `Regions` 插件，用于拖拽修剪选区)。
> * **音频处理引擎 (Node.js 端)**: `fluent-ffmpeg` 搭配 `ffmpeg-static`。前端只负责预览，实际的修剪、调音量、变速、合并和导出操作，统交由 Electron 主进程调用 ffmpeg 完成，以保证极致的稳定性和导出速度。
> 
> 
> ### 界面设计 (UI) 要求
> 
> 
> 参考现代深色系风格（Dark Mode）：
> 1. **布局**: 窗口默认 1024x768，支持全屏和自适应缩放。
> 2. **左侧工具栏**: 包含 [✂️ 修剪]、[🔊 音量]、[⏱️ 变速]、[🔗 合并] 四个按钮，点击后在底部控制台切换显示对应的参数调节滑块。
> 3. **中央波形区**: 占据主视觉。使用 Wavesurfer 渲染波形（波形颜色为青绿色 `#00d0bf`）。必须挂载 Regions 插件，允许用户在波形上自由拖拽起始和结束边界。
> 4. **底部控制台**: 包含 [播放/暂停] 按钮、当前时间显示，以及右下角醒目的 [导出保存] 按钮。
> 
> 
> ### 核心功能实现逻辑 (请严格按照此逻辑编写代码)
> 
> 
> 1. **文件导入**:
> * 点击导入按钮，通过 `dialog.showOpenDialog` 选择本地文件。
> * 将文件路径传递给渲染进程，使用 `wavesurfer.load()` 渲染波形。
> 
> 
> 2. **实时预览 (Frontend)**:
> * **音量和变速**: 用户拖动滑块时，直接调用 `wavesurfer.setVolume()` 和 `wavesurfer.setPlaybackRate()` 实现实时试听预览。
> * **修剪**: 利用 Regions 插件框选。播放时通过监听时间限制在选区内播放。
> 
> 
> 3. **最终导出与合并 (Backend / IPC)**:
> * 点击 [导出保存] 时，渲染进程将用户设定的所有参数打包（如：`{filePath: '...', trimStart: 1.2, trimEnd: 4.5, volume: 1.5, speed: 1.2, mergeFilePath: '...'}`）通过 `ipcRenderer.invoke` 发送给主进程。
> * **主进程逻辑**: 接收参数后，调用 `fluent-ffmpeg`。构建 ffmpeg 命令链：先截取 (trim)，再添加音频滤镜处理音量 (volume) 和变速 (atempo)，最后如果有合并文件，执行 concat。最后输出为新的 `.mp3` 或 `.wav` 文件。
> 
>
